## 关键技术决策

### v1.1.1
- **1 runner 1 进程**: 简化并发模型, 用 state 文件作为 IPC
- **子进程 + log 捕获**: 用 `subprocess.Popen(stdout=PIPE)` + 后台 thread 转发到 log
- **PIPELINE marker**: 在 stdout 打 `[PIPELINE] book=X ch=Y stage=Z status=W`, dashboard 解析
- **隐式 stage context**: `llm.set_stage_context(stage, ch)`, 后续 complete/call 自动带
- **SSE 优于 WS**: 单向流足够, 浏览器原生 EventSource 简单

### v1.3
- **会话过渡三件套**: entity_diff (实体变化追踪) + review_actions (自动修订) + session_log (JSONL 结构化日志)
- **pipeline_v2**: 8 阶段 pipeline + checkpoint/recover/resume state machine
- **LLM provider 三段链**: config_loader → get_llm(book=book) 透传 book 上下文 → health API
- **refactor M1/M2**: 统一 _base.html + main.css + common.js 基础设施, 全部 alert/confirm 收敛到 NW 命名空间
